# 学生端功能及交互方案

## 0. 总览

### 0.1 产品定位

学生端不是“作业后台”，而是 K12 英语学习的每日陪伴入口。

它的第一目标不是展示所有功能，而是让孩子每天以最低认知成本完成一个高质量学习闭环：

```text
打开 App
-> 看到今日主线
-> 点击整张卡片进入
-> 完成第一题
-> 获得即时反馈
-> 继续完成今日学习
```

### 0.2 北极星体验

学生每天打开 App 的前 30 秒必须保持极简、聚焦、正反馈。

- 首页首屏只服务 `今日主线`。
- `今日主线` 是唯一第一行动。
- 录音完成后立即给视听反馈，但不承诺 AI 评分瞬间完成。
- AI 评分可异步返回，不能阻塞孩子继续做题。
- 普通消息、商城、奖励、学习地图、活动红点不得抢占首屏注意力。

### 0.3 总体信息架构

学生端固定为 3 个一级区。

| 一级区 | 路由 | 定位 | 内容 |
| --- | --- | --- | --- |
| 首页 | `/home` | 今日学习探索页 | 今日主线、听说写玩、学习地图、点评入口、成长状态 |
| 作业 | `/activities` | 作业与课程中心 | 今日任务、全部作业、待补做、课程安排、作业详情 |
| 个人中心 | `/profile` | 极简个人工具 | 星币、消息、设置、关于 |

一级导航在平板和手机横屏都必须可见，只允许形态变化，不允许功能缩水。

### 0.4 当前冲突与裁定

| 冲突点 | 裁定 |
| --- | --- |
| 首页第一屏功能过多 | 第一屏只服务今日主线，其他探索能力进入第二屏或弱入口 |
| `听说写玩` 与 `学习地图` 重叠 | `听说写玩` 按能力组织，`学习地图` 按教材/资源路线组织 |
| `自然拼读` 放哪里 | 教材入口在学习地图，错词/小游戏素材可进入 `玩` |
| 点评页是否提供再练一次 | 点评页只做听觉对比和纠错，不提供练习入口；练习回作业流或错音本 |
| 作业执行页是否保留左右分栏 | 废除左右分栏，改为一屏一题沉浸舞台 |
| 热区点读是否作为默认主流程 | 默认顺序跟读，热区点读作为特定题型模式 |
| 个人中心是否承载家长通、成就、书架 | 不承载，只保留星币、消息、设置、关于 |
| 星币是否可兑换实物 | 不可兑换现实物品，只做应用内虚拟消耗 |
| AI 失败是否暴露技术错误 | 不暴露，全部转为儿童友好降级话术 |
| 页面是否可各自写样式 | 不允许，必须经过设计 token 和统一组件 |

### 0.5 总体视觉方向

- 使用天蓝 / 蓝绿渐变作为全局学习场景背景。
- 使用高饱和黄色、绿色、橙色作为行动和反馈色。
- 保持大圆角、悬浮卡片、柔和阴影、卡通插画。
- 拆除后台式大白框、表格表头、密集字段和固定分栏。
- 高频学习场景采用无界视觉，让卡片直接漂浮在背景上。

## 1. 分述：核心信息架构

### 1.1 首页

首页只回答一个问题：孩子今天先做什么？

首页第一屏的主视觉只服务 `今日主线`。其他入口只允许以弱露出或状态徽章出现，不得抢占 C 位，也不得弹出红点或运营浮层打断孩子进入作业。

- 核心入口：`今日主线`
- 弱露出：`听 / 说 / 写 / 玩`
- 弱入口：`点评中心`
- 状态徽章：`星币 / 连对 / 成长状态`
- 轻量入口：`学校与班级动态`

`学习地图`、`积分兑换`、运营活动和普通消息不进入第一屏主视觉。

首页第二屏保留：

- `学习地图`
- `3 天内历史作业补做`
- `自然拼读`
- `国家地理 PM`
- `积分兑换`
- 极简个人中心入口：`星币 / 消息 / 设置 / 关于`

首页横屏分页交互规则：

- 首页只保留左右滑动作为翻页方式，不再显示上一页、下一页、回首页这类显性按钮。
- 左滑进入第二屏，右滑回到第一屏；分页指示只做弱提示，不承载点击操作。
- 从第二屏的 `听说写玩`、`学习地图`、`补星计划`、`自然拼读`、`国家地理 PM`、`魔法商店` 等入口进入子页面后，左滑回退必须回到首页第一屏，而不是停留在第二屏。
- 子页面不要新增“回首页”按钮；如必须退出，使用系统返回或页面左上返回，返回目标仍然是首页第一屏。
- 作业列表、学习地图等内容流页面不放顶部刷新按钮，统一使用下拉刷新。

### 1.2 作业

作业页负责所有作业相关能力：

- 今日作业
- 全部作业
- 待完成
- 已完成
- 待补做
- 课程安排
- 作业详情
- 作业练习舞台

`今日任务`、`任务中心`、`课程表` 不再作为首页独立弹窗维护，统一收口到 `/activities`。

### 1.3 个人中心

个人中心只保留低频工具：

- 星币
- 消息
- 设置
- 关于

个人中心不承载作业、课程、点评、家长通、动态等高频学习入口。

## 2. 分述：首页体验

### 2.1 今日主线

`今日主线` 是首页 C 位，是孩子每天进入学习的舞台入口。

交互规则：

- 整张卡片都是点击热区。
- 卡片内部不放需要精准点击的小按钮。
- `继续第 1 句` 可以作为提示文案，但不能成为唯一按钮。
- 移除右下角小书本图标。
- 星币移出卡片内部，避免干扰主行动。
- 点击后有水波纹、轻微按压、阴影变化或吉祥物回应。

状态规则：

| 状态 | 主文案 | 点击行为 |
| --- | --- | --- |
| 有未完成作业 | `继续今天作业` | 进入未完成第一题或最近进度 |
| 今日已完成 | `今天完成啦` | 进入结算、错音回顾或推荐复习 |
| 今日暂无作业 | `今天没有新作业` | 推荐阅读、自然拼读或错音本 |
| 作业已过期 | `这份作业已截止` | 进入详情或提示联系老师 |
| 离线可继续 | `继续上次学习` | 进入本地缓存进度，后台静默同步 |

空间转场：

- 起点是首页今日主线大卡。
- 点击后卡片渐变背景扩展为作业舞台背景。
- 首页文字淡出，作业题目和底部控制舱入场。
- 吉祥物从首页移动到作业页角落，成为伴学小教练。
- 返回时视觉上回到今日主线卡片。

### 2.2 听说写玩

`听说写玩` 是能力健身房，按动词和技能组织。

| 模块 | 定位 | 内容形态 | 交互规则 |
| --- | --- | --- | --- |
| 听 | 沉浸式磨耳朵 | 儿歌、绘本原声、自然拼读动画、课堂音频 | 不强制跟读，不默认打分 |
| 说 | 情景化开口训练 | AI 角色扮演、情景对话、视频配音 | 弱化分数，鼓励表达 |
| 写 | 线上线下多模态桥梁 | 拍照上传、字母描红、作品展示、拍照识物 | 强调作品感，不做表单感 |
| 玩 | 星币消费与薄弱点复习 | 错词消消乐、翻翻卡、装扮、吉祥物养成 | 优先复用错词和薄弱点 |

规则：

- 不展示 `还剩 X 项` 这类催促文案。
- 使用动词化卡片文案，例如 `听一首歌`、`和 AI 说两句`、`上传作品`、`玩错词小游戏`。
- `听 / 写` 可偏冷色，`说 / 玩` 可偏暖色。
- 从 `听说写玩` 进入的落地页返回时，必须清理首页分页状态并回到首页第一屏，避免孩子回到第二屏后迷失。

### 2.3 学习地图

`学习地图` 是长期路线图，按教材、资源、路线组织。

它不叫 `今日学习地图`，也不承担今日任务压力。

学习地图四类入口：

| 入口 | 推荐文案 | 定位 |
| --- | --- | --- |
| 历史补做 | `补星计划` / `时光救援` | 补做 3 天内未完成或需补救的作业 |
| 自然拼读 | `自然拼读` | Phonics 输入与游戏化巩固 |
| 国家地理 PM | `国家地理 PM` | 高质量分级阅读 |
| 积分兑换 | `魔法商店` / `星币盲盒` | 星币消费和奖励兑换 |

视觉规则：

- 删除学习地图外层大白框。
- 卡片直接漂浮在全局渐变背景上。
- 封面卡片允许横向截断，露出下一张 15% 到 20%。
- 学习地图不展示任务剩余数。
- 学习地图内的日历、筛选、状态类按钮使用轻量胶囊样式，高度低于主行动按钮，避免抢占视觉 C 位。

### 2.4 学校与班级动态

左上角入口从通用 `英语学习` 改为学生当前上下文。

入口文案：

- 完整：`XX学校 XX班级动态 · 英语`
- 空间不足：`XX班级动态 · 英语`
- 极窄：学校 logo / 班级徽章 + `班级动态`

点击后只展示轻量弹窗，不进入独立页面。

弹窗内容：

- 老师留言
- 班级荣誉
- 学校简介

学生端不提供班级切换。`school_id`、`class_id`、学科上下文跟随学生账号或登录身份固定。

## 3. 分述：作业体验

### 3.1 作业入口收口

| 来源 | 目标 |
| --- | --- |
| 首页今日主线 | `/activities/:activityId` 或最近题目 |
| 作业一级入口 | `/activities` |
| 课程表 | `/activities?tab=schedule` |
| 作业列表卡片 | `push('/activities/:activityId')` |

### 3.2 作业详情分层

作业详情不再把所有功能挤在同一个页面里，而是拆成三层：

| 层级 | 目标 | 主要内容 |
| --- | --- | --- |
| 作业列表层 | 先让孩子知道今天要做几项 | 任务卡片、完成状态、教材名称、做任务按钮 |
| 绘本舞台层 | 以课本/绘本为主，孩子专注当前页 | 大图、句子描边、点读录音入口、完成按钮 |
| 反馈确认层 | 每句录完立刻形成闭环 | 自动播放标准读音、播放我的发音、分数、重录/继续 |

进入 `/activities/:activityId` 时默认展示作业列表层。孩子点击某一项 `做任务` 后，再进入绘本舞台层。

### 3.3 作业练习舞台

作业执行页采用“绘本优先 + 点读录音”，废除左右分栏和后台式状态堆叠。

顶部：

- 左侧退出或回到作业列表。
- 中间图形化进度，例如星星、脚印、能量格。
- 右侧低频设置。
- 不显示 `当前第 1/1 句`、`已完成 0/1`、`待完成` 这类报表标签。

中部：

- 教材图、句子或题面占据视觉中心。
- 图片不使用厚重边框和多层阴影。
- 英文句子用大字号悬浮显示。
- 点击 `点读录音` 后，课本上的句子/热区以描边形式显示。
- 每个描边句子都可点击；点击后先播放标准读音，再进入该句录音状态。
- 热区题只显示呼吸锚点或轻量描边，不遮挡绘本主体。

底部控制舱：

- 左侧：`听原音`
- 中央：麦克风录音主按钮
- 右侧：`下一题` / `继续`
- 全部句子完成后，显示 `完成并提交` 主按钮。

### 3.4 题型模式

| 模式 | 适用题型 | UI 形态 | 默认性 |
| --- | --- | --- | --- |
| 听力播放器模式 | 纯听力、儿歌、磨耳朵 | 唱片机或封面播放器 | 按题型触发 |
| 顺序跟读模式 | 单句跟读、逐句复读、配音 | 大图/句子 + 底部录音控制舱 | 低龄默认主流程 |
| 热区点读模式 | 教材热区、点图读句 | 教材图全屏 + 呼吸热区 | 特定题型 |
| 分页录音模式 | 整页朗读 | 页面大图 + 分页进度 | 较长文本 |

默认主流程使用 `顺序跟读模式`。热区点读作为能力保留，但不替代默认主流程。

### 3.5 录音与反馈

录音交互：

- 单击麦克风开始。
- 录音中显示波纹和计时。
- 再次单击结束。
- 录音结束后立即弹出反馈确认层。
- 反馈层先自动播放标准读音，再允许播放我的发音。
- 允许试听、重录、继续下一句。
- 当前句全部完成后，本地记录完成状态；整份作业最后再提交。

反馈底栏：

- 正确：绿色底板、欢快音效、星币动效。
- 需改进：橙色 / 红色底板，给具体建议。
- 发音问题：标出词或音素，提供重听和再试一次。

这里的 `再试一次` 只属于当前题即时反馈，不等同于点评详情页的 `再练一次` 入口。点评页负责回放、诊断和纠错理解，不承担重新做题。

## 4. 分述：点评中心

### 4.1 点评列表

点评中心不采用表格形态。

列表卡片规则：

- 左侧使用作业封面、教材截图或题目插图。
- 不再使用统一黄色音乐图标。
- 主标题展示作业或题目名称。
- 副信息展示课程、老师、时间。
- 未读卡片使用浅蓝底、红点和强调按钮。
- 已读卡片使用白底和弱化按钮。
- 进入详情并停留 1 到 2 秒后再标记已读。

### 4.2 作业回放

进入点评后先回放当时表现。

- 进入页面自动播放学生录音。
- 主区域展示大图、教材插画或句子。
- 底部集中播放控制舱：播放/暂停、进度条、听原音、听我的。
- 右侧保留强入口：`发音小报告`。

### 4.3 查看点评

查看点评页不是文字报告，而是听觉对比和纠错闭环。

结构：

- 顶部：分数、星星、徽章、鼓励语。
- 中部：能力摘要，可用雷达图或能力卡。
- 核心：按单词展示红黄绿纠错文本。

点击红色或黄色单词，只弹出：

- `听原音`
- `听我的`

不提供 `再练一次`。练习入口回到作业练习流或错音本。

文字点评默认折叠，按钮为 `展开老师寄语`。

## 5. 分述：个人中心、家长与学校

### 5.1 个人中心

个人中心只保留：

- 星币
- 消息
- 设置
- 关于

### 5.2 家长信任层

家长空间不作为个人中心一级入口，避免破坏学生端的极简工具定位。它收纳在 `设置` 或 `关于` 的受保护入口中，并使用算术锁或家长验证。

展示内容：

- 今天开口读了多少句。
- 今天最棒的一句录音。
- 老师点评摘要。
- 本周高光时刻。
- 使用时长和休息记录。

分享规则：

- 分享默认隐藏真实姓名。
- 含录音分享必须家长主动确认。
- 学校可配置是否允许分享。
- 分享不得暴露账号、手机号、精确班级和敏感时间线。

### 5.3 学校品牌与首启

学校品牌贯穿欢迎页、登录页、首页学校入口。

管理端配置：

- 学校名称
- 学校 logo
- 主色 / 辅助色
- 欢迎标题
- 欢迎副标题
- 登录页提示语
- 家长提示语
- 帮助入口

教师端需要能预览学生端首页、欢迎页和班级动态效果。

## 6. 分述：登录与身份路由

登录页是身份路由中心，而不只是账号密码输入。

支持：

- 账密登录
- 手机号 + 验证码
- 家长微信扫码授权

规则：

- 账密登录自动保存用户名。
- 密码不默认明文。
- 验证码调起数字键盘。
- 多孩子账号登录后先展示大头像身份选择。
- Token 建议有效期约 30 天。
- Token 未过期时无感进入上次孩子首页。

异常状态：

- Token 过期：回登录页，保留学校和用户名。
- 学校配置失效：回学校码或学校选择。
- 微信未绑定孩子：进入绑定流程。
- 网络弱：允许展示缓存学校品牌，但不允许离线登录未验证账号。

## 7. 分述：视觉与组件系统

### 7.1 设计原则

- 明亮、饱和、卡通，但不杂乱。
- 大圆角、大点击区、少文字。
- 高频学习场景使用无界背景和悬浮卡片。
- 低频工具场景保持同一套圆角、阴影、按钮和图标体系。
- 页面不得自行创造新样式。

### 7.2 Design Tokens

统一维护：

- `StudentColors`
- `StudentRadii`
- `StudentSpacing`
- `StudentTypography`
- `StudentShadows`
- `StudentMotion`

## 8. 分述：上线级体验护栏

### 8.1 星币经济

产出：

- 今日主线是最高收益来源。
- 听说写玩少量产出，并收益递减。
- 连续点击、重复播放、无效录音不产出星币。

消耗：

- 只用于应用内虚拟消耗。
- 不兑换现实物品。
- 不影响作业完成。

透明：

- 家长可查看星币账单。
- 账单展示来源、去向和时间。

### 8.2 AI 评审降级

| 场景 | 学生端表现 | 处理 |
| --- | --- | --- |
| 等待超过 2 秒 | `老师正在认真听你的发音哦` | 等待异步结果 |
| 网络失败 | `网络小精灵迷路啦，我们先做下一题` | 本地同步队列 |
| 声音太小 | `没听清，可以再大声一点吗` | 可重录 |
| 结果迟到 | 学生继续做题 | 静默更新点评和错音本 |

禁止暴露工程文案，例如 `Error`、`timeout`、`request failed`。

### 8.3 健康与可访问性

- 连续使用达到配置时长后触发休息。
- 默认建议 20 分钟。
- 学生侧不可轻易跳过。
- 家长可通过家长锁处理受控例外。
- 批改结果不只依赖红绿颜色，同时提供符号、形状或下划线。
- 平板核心热区不低于 `64x64 dp`。
- 横屏手机避开刘海、系统导航和边缘手势区。
- 系统开启减少动态效果时关闭复杂转场。

### 8.4 工程治理

- 页面层禁止硬编码业务样式。
- 业务主按钮使用封装按钮。
- 用户可见文本优先使用封装文本。
- 星币、红点、空状态、弹窗、底部控制舱使用统一组件。
- 允许基础布局组件，但不能绕过组件系统直接拼业务 UI。
- CI / Lint 检查硬编码样式、原生主按钮、未接入页面骨架等问题。

## 10. 分述：技术实现路径

### 10.1 响应式

- 使用 `responsive_builder` 或现有断点能力区分手机横屏和平板。
- 手机横屏和平板共用功能结构，只调整比例、字号、密度。
- 所有主界面保留 `SafeArea`。
- 横向画廊使用 `PageView(viewportFraction)`，不使用硬编码宽高的 `Row + Container`。

### 10.2 录音与 AI

`AudioRecordButton` 状态机：

```text
Idle -> Recording -> Processing -> Done
```

非阻塞流程：

```text
录音完成
-> 本地保存答题状态
-> 音频上传 Supabase Storage
-> 进入下一题或轻量处理中
-> 后台等待 AI / Python Agent
-> 结果回来后更新点评、错音本
```

### 10.3 离线优先

本地需要保存：

- 当前作业 ID
- 当前题号
- 每题答题状态
- 本地录音暂存路径
- 待上传队列

App 意外关闭后，下次打开提示继续上次学习。未上传录音后台静默补传。

## 11. 分述：返回、弹窗与验收

### 11.1 返回规则

| 场景 | 推荐方式 |

### 11.4 验收标准

核心验收：

- 首页首屏只服务今日主线。
- 今日主线整卡可点击。
- 听说写玩和学习地图边界清晰。
- 学习地图采用无界视觉。
- 个人中心只保留星币、消息、设置、关于。
- 作业执行页不再使用左右分栏。
- 默认顺序跟读，热区点读只作为题型模式。
- 点评页只做听觉对比和纠错，不提供再练入口。
- 星币不可兑换现实物品。
- AI 异常全部儿童友好降级。
- 家长分享默认隐私保护。
- 组件、token、页面骨架统一。
- 关键能力支持灰度与回滚。
- 回滚不丢本地进度和待上传录音。
